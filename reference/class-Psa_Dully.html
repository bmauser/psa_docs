Class Psa_Dully

Simple template engine class.

I called this class Dully to be opposite to Smarty. It has few methods that are similar to those in Smarty template engine.

All that Dully does is to put values from associative array into the local namespace and includes .php (template) file. There are no special template tags, just use PHP code blocks to echo values. Templates are ordinary PHP files and you can write them as any other .php script, but keep in mind that the point of template engine should be to separate business logic from presentation. So you should not put any logic (calculations, getting data from a database ...) into templates. With Smarty that is easier to achieve and it has many advanced features so I suggest that you use Smarty as a template engine. If you need simple and fast template engine or don't want to learn Smarty's tags you can use Dully. Dully class doesn't include anything from PSA so you can use this class as template engine in any PHP application.

This class is inspired from here and there are some interesting thoughts about template engines.


Usage examples:


1) Simple example with one template file:

Template file template.tpl:

This is text from template file template.php<br/>
My car is <?php echo $my_car_color ?>.<br/>
Something else: <?php echo $some_name ?>

.php file:

<?php

// make a new Psa_Dully object
$Psa_Dully = new Psa_Dully('path/to/dir/with/templates');

// assign some data for template
$Psa_Dully->assign('my_car_color', 'Black');
$Psa_Dully->assign('some_name', 'bla bla bla');

// display fetched template
$Psa_Dully->display('template.tpl');

?>

The printout of the above .php script will be:

This is text from template file template.php
My car is Black.
Something else: bla bla bla

Note: you can write <?= $some_name ?> instead of <?php echo $some_name ?>. Use of this shortcut requires short_open_tag PHP ini option to be on.


2) You can put one rendered template into another:

template file template1.tpl:

This is text from template1.php<br/>
<?php echo $var1 ?><br/>
<?php echo $var2 ?>

template file template2.tpl:

This is text from template2.php

.php file:

<?php

// make a new Psa_Dully object
$Psa_Dully = new Psa_Dully('path/to/dir/with/templates');

// put the content of fetched template template2.tpl in the variable $var1
$var1 = $Psa_Dully->fetch('template2.tpl');

// assign values for var1 and var2
$Psa_Dully->assign('var1', $var1);
$Psa_Dully->assign('var2', 'bla bla bla');

// display template template1.tpl
$Psa_Dully->display('template1.tpl');

?>

The above example will output:

This is text from template1.php
This is text from template2.php
bla bla bla


3) In this kind of templates it's nice to use alternative PHP syntax for control structures. See details in PHP documentation.

<? if ($variable == 'abcd'): ?>
        <h1>this is something</h1>
        <p>
                bla bla
        </p>
<? endif ?>

<? foreach ($array as $value): ?>
        <div><?= $value['email']) ?></div>
        <div><?= $value['phone_number']) ?></div>
<? endforeach ?>
