Class Psa_Hook_Before_User_Create

Abstract class for Psa_Hook_Before_User_Create hooks.

You can create a new user by passing 'new' to Psa_User class constructor. See Psa_User::__construct() method of Psa_User class.

Before a new user is to be created with Psa_User::save() method, psa_main() method of the hook object that extends this class will be called. You can write Psa_Hook_Before_User_Create hooks by extending this class.

Example:

<?php
class my_hook extends Psa_Hook_Before_User_Create{

    // you have to define psa_main() method in your child class
    function psa_main($user){

        // set some property to new user
        $user->favorite_background_color = 'white';
    }
}
?>

When this example hook is registered, every new user will have property called favorite_background_color with value 'white'.

Note: this hook is called before the new user is created and saved to the database. Because of that, this hook will be called even if the user is not created due to some error, ex. user with the same userneme already exists in the database. If you set some property to the user like favorite_background_color in the example above and the user is successfully created, this property will be saved to the database and you don't have to call Psa_User::save() method later to store it. That is unlike Psa_Hook_After_User_Create hook. Also, a user ID is unknown when this hook is called and is set after the user is created.
